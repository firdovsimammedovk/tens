TT-NN Overview
##############

.. contents:: Sidebar
   :local:
   :depth: 2

Introduction
============

TT-NN is the primary programming interface for running neural network workloads on Tenstorrent hardware. It provides a high-level API for working with tensors, executing operations, and building machine learning models. TT-NN abstracts many low-level details of hardware execution while still allowing efficient utilization of the Tenstorrent architecture.

What TT-NN Provides
===================

TT-NN enables developers to:

* Create and manipulate tensors
* Execute neural network operations
* Run machine learning models
* Integrate with existing frameworks such as PyTorch

The API is designed to resemble familiar deep learning frameworks while exposing capabilities specific to Tenstorrent hardware.

Core Concepts
=============

TT-NN revolves around several key concepts:

Tensors
-------

Tensors represent multidimensional arrays that store model data and intermediate results.

Operations
----------

Operations perform computations on tensors. Examples include:

* Matmul
* Convolution
* Softmax
* Layer normalization

Devices
-------

A device represents a Tenstorrent accelerator capable of executing neural network workloads. TT-NN operations are executed on one or more devices.

How TT-NN Fits into the Stack
==============================

TT-NN sits between the model compiler and the device runtime.

::

    framework
        ↓
      TT-Forge
        ↓
       TT-NN
        ↓
    TT-Metalium
        ↓
      hardware

It translates neural network operations into executable workloads that are scheduled by the runtime.

Typical Workflow
================

A typical TT-NN workflow includes:

1. Initialize device
2. Create tensors
3. Execute operations
4. Retrieve results

Example:

::

    import ttnn

    device = ttnn.open_device()
    a = ttnn.from_torch(tensor_a)
    b = ttnn.from_torch(tensor_b)
    result = ttnn.matmul(a, b)

When to Use TT-NN
=================

TT-NN is recommended for:

* Running machine learning models
* Implementing neural network operations
* Developing inference workloads

Developers who need direct control over kernel execution or hardware resources may use TT-Metalium instead.
