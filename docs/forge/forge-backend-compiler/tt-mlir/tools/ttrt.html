ttrt

Tenstorrent Runtime - Execute compiled TT-MLIR binaries on hardware.

Overview

ttrt (Tenstorrent Runtime) executes compiled TT-MLIR programs on Tenstorrent hardware. It handles device initialization, memory management, and kernel execution.

Basic Usage

ttrt <command> [options]

Commands

execute

Execute a compiled model:

ttrt execute model.ttnn [--input input.bin] [--output output.bin]

Options:

  • --input <file> - Input tensor file (binary format)

  • --output <file> - Output tensor file (binary format)

  • --device <id> - Device ID (default: 0)

  • --verify - Verify results against reference

  • --iterations <n> - Run N iterations for benchmarking

query

Query device information:

ttrt query [--device <id>]

Output:

  • Device architecture

  • Number of cores

  • Memory capacity

  • PCIe information

Save artifacts:

ttrt query --save-artifacts

This saves system_desc.ttsys for use during compilation.

reset

Reset device:

ttrt reset [--device <id>]

info

Display TTRT version and configuration:

ttrt info

Common Workflows

Basic Execution

Execute a model with default inputs:

ttrt execute model.ttnn

With Custom Inputs

# Prepare input
python prepare_input.py --output input.bin

# Execute
ttrt execute model.ttnn --input input.bin --output result.bin

# Process output
python process_output.py --input result.bin

Benchmarking

Run multiple iterations for timing:

ttrt execute model.ttnn --iterations 100

Output shows:

  • Average execution time

  • Min/max times

  • Throughput

Device Management

# Query all devices
tt-smi

# Get device info
ttrt query --device 0

# Reset if needed
ttrt reset --device 0

Multi-Device Execution

Execute on specific device:

# Device 0
ttrt execute model.ttnn --device 0

# Device 1
ttrt execute model.ttnn --device 1

Input/Output Format

Binary Tensor Format

Tensors are stored as raw binary data:

import numpy as np

# Save tensor
tensor = np.random.randn(32, 128).astype(np.float32)
tensor.tofile('input.bin')

# Load tensor
output = np.fromfile('output.bin', dtype=np.float32)
output = output.reshape(32, 128)

Environment Variables

Logging

# Enable debug logging
export TT_METAL_LOGGER_LEVEL=DEBUG

# Log specific components
export TT_METAL_LOGGER_TYPES=Op,Device,Runtime

Profiling

# Enable profiling
export TT_METAL_PROFILE=1

# Profile memory
export TT_METAL_PROFILE_MEMORY=1

# Profile ops
export TT_METAL_PROFILE_OP=1

Cache

# Enable kernel cache
export TT_METAL_CACHE_PATH=/path/to/cache

Debug

# Enable kernel debugging
export TT_METAL_KERNEL_DEBUG=1

# Dump tensors
export TT_METAL_DUMP_TENSORS=1

Python API

Execute from Python:

import ttrt

# Load model
model = ttrt.load("model.ttnn")

# Execute
output = model.execute(input_tensor)

# Multiple inputs
outputs = model.execute([input1, input2])

With options:

# Benchmarking
result = model.execute(input, iterations=100)
print(f"Avg time: {result.avg_time_ms} ms")

# Profiling
result = model.execute(input, profile=True)
result.print_profile()

Performance Profiling

Enable Profiling

export TT_METAL_PROFILE=1
ttrt execute model.ttnn

View Profile Data

python -m tt_metal.tools.profiler.process_ops_logs

Output includes:

  • Op-by-op execution time

  • Memory bandwidth usage

  • Kernel execution time

  • DMA transfer time

Visualize Profile

python -m tt_metal.tools.profiler.visualize \
  --input profile.json \
  --output timeline.html

Troubleshooting

Device Not Found

Symptom: Error: No devices found

Solutions:

  1. Check device status:

    tt-smi

  2. Verify drivers:

    lsmod | grep tenstorrent

  3. Reset device:

    ttrt reset

Execution Fails

Symptom: Runtime errors during execution

Debug:

# Enable debug logging
export TT_METAL_LOGGER_LEVEL=DEBUG
ttrt execute model.ttnn

Out of Memory

Symptom: Error: Out of memory

Solutions:

  1. Check memory usage:

    tt-smi -m

  2. Reduce batch size

  3. Use DRAM instead of L1

  4. Enable memory profiling to identify leaks

Incorrect Results

Symptom: Wrong output values

Debug:

  1. Dump intermediate tensors:

    export TT_METAL_DUMP_TENSORS=1
ttrt execute model.ttnn

  2. Verify against reference:

    ttrt execute model.ttnn --verify

  3. Check input data format

Advanced Usage

Custom Memory Configuration

import ttrt

config = ttrt.MemoryConfig(
    l1_size=1024*1024,  # 1MB L1
    dram_channels=12
)

model = ttrt.load("model.ttnn", memory_config=config)

Multi-Device Execution

import ttrt

# Load on multiple devices
model0 = ttrt.load("model.ttnn", device_id=0)
model1 = ttrt.load("model.ttnn", device_id=1)

# Execute in parallel
import concurrent.futures

with concurrent.futures.ThreadPoolExecutor() as executor:
    future0 = executor.submit(model0.execute, input0)
    future1 = executor.submit(model1.execute, input1)

    result0 = future0.result()
    result1 = future1.result()
