Target Independence

* Custom dialects only for TT-specific operations

Reusability

* Easy to extend with new operations or optimizations
* Passes can be added without modifying existing code
* Each dialect is independent and composable

TT-MLIR is designed with modularity in mind:

~~~~~~~~~~
Modularity

* Transformations preserve semantics across levels
* Lower levels expose hardware-specific details
* Higher levels are easier to optimize at a coarse grain

Each dialect level progressively lowers abstractions:

~~~~~~~~~~~~~~~~~~~~
Progressive Lowering

-----------------
Design Principles

* **Optimization**: Register allocation, scheduling, hardware-specific tuning
* **Operations**: Direct hardware primitives
* **Purpose**: Low-level hardware operations

~~~~~~~
TTMetal

* **Optimization**: Memory layout optimization, tile sizing, sharding
* **Operations**: Optimized NN kernels specific to TT hardware
* **Purpose**: Neural network optimized operations

~~~~~~~~~~~~~~~~~~~~~~~~~
TTNN (TT Neural Network)

* **Optimization**: Graph-level optimizations, fusion, constant folding
* **Operations**: Standard ML operations (matmul, conv, relu, etc.)
* **Purpose**: High-level, hardware-agnostic operations

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
TTIR (TT Intermediate Representation)

TT-MLIR uses three primary dialect levels:

---------------------
Multi-Level IR Design

   TTRT Runtime Execution
            ↓
   Flatbuffer Binary (.ttnn)
            ↓
   TTMetal Code Generation
            ↓
      TTMetal Dialect (Low-Level IR)
            ↓
   TTNN Optimization Passes
            ↓
       TTNN Dialect (NN-Optimized IR)
            ↓
   TTIR Optimization Passes
            ↓
       TTIR Dialect (High-Level IR)
            ↓
   Frontend (PyTorch/ONNX/JAX)

The compilation process follows this general flow:

----------------
Compilation Flow

* **System Description** - Hardware configuration and capabilities
* **Runtime** - Execute compiled programs on hardware
* **Passes** - Transform and optimize IR at each level
* **Dialects** - Define operations and types at different abstraction levels
