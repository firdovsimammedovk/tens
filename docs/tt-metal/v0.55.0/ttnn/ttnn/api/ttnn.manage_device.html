ttnn.manage_device

ttnn.manage_device(device_id: int) → MeshDevice

Open a device and automatically close it when leaving a context block.

Signature 

ttnn.manage_device(device_id: int) -> ContextManager[ttnn.device.Device]

Description

Creates a context manager for safe device lifecycle handling. The device is opened when entering the with block and closed on exit, including exception paths.

This is the recommended pattern for most scripts because it prevents leaked device handles when failures happen mid-execution.

Parameters

  • device_id: int - Numeric device identifier passed to ttnn.open_device().

Returns

  • ContextManager[ttnn.device.Device] - Yields the opened device inside the with block.

Raises

  • RuntimeError - Propagated from ttnn.open_device() if the device cannot be opened.

  • RuntimeError - Propagated from ttnn.close_device() if close fails during context teardown.

Notes

  • The device is always closed in a finally block.

  • Exceptions raised inside the user block are not swallowed and continue to propagate.

  • Prefer this API over manual open/close pairs to avoid resource leaks.

Examples

Basic usage with automatic cleanup:

>>> import ttnn
>>> with ttnn.manage_device(device_id=0) as device:
...     tensor = ttnn.zeros((2, 3), device=device)
...     print(tensor.shape)

Equivalent manual lifecycle (not recommended for most code):

>>> import ttnn
>>> device = ttnn.open_device(device_id=0)
>>> try:
...     output = ttnn.ones((4, 4), device=device)
... finally:
...     ttnn.close_device(device)

See Also

  • ttnn.open_device - Open a device explicitly without context management.

  • ttnn.close_device - Close and release a previously opened device.

  • ttnn.synchronize_device - Synchronize queued work before teardown when needed.
